How I Work

I optimize for systems that are clear to operate, safe to change, and honest about what they can and can’t do.

Anchor on the problem and the data before picking a model or tool.

Design modular pipelines first—complexity only when it clearly earns its place.

Treat LLMs as systems: retrieval, guardrails, latency, monitoring—not a single API call.

Make quality measurable: evals, monitoring, and explicit failure paths.

Build for real users, scale, and ops constraints—not idealized notebooks.
